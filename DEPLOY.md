# Деплой и обратная связь

## Почему не отправляется форма на сервере

Форма обратной связи шлёт данные на **API**, адрес которого задаётся при **сборке** фронта переменной `VITE_API_BASE_URL`. Если при деплое вы не задали этот URL или оставили `http://localhost:4000`, в браузере посетителя запросы уходят на его же «localhost» — до вашего сервера они не доходят.

---

## Что сделать

### 1. У вас есть свой бэкенд (Node и т.д.)

- На **сервере**, где собираете фронт (`npm run build`), создайте/отредактируйте `.env`:
  ```env
  VITE_API_BASE_URL=https://ваш-домен-с-api.com
  ```
  Без слэша в конце. Пример: `https://api.ionstudio.ru` или `https://ionstudio.ru` (если API за прокси на том же домене).

- Соберите проект **после** сохранения `.env`:
  ```bash
  npm run build
  ```
  Дальше деплойте содержимое папки `dist`.

- На **бэкенде** включите CORS для домена фронта. Разрешите запросы с того origin, где открыт сайт, например:
  - `https://ionstudio.ru`
  - И заголовки: `Content-Type`, при необходимости `Authorization`.

  Иначе браузер заблокирует ответ и форма будет «не отправляться».

### 2. Бэкенда нет — только статический сайт

Можно использовать сервис форм (Formspree, Getform и т.п.):

1. Зарегистрируйтесь на [Formspree](https://formspree.io), создайте форму и скопируйте URL вида `https://formspree.io/f/xxxxx`.
2. При сборке задайте в `.env`:
   ```env
   VITE_CONTACT_FORM_ENDPOINT=https://formspree.io/f/xxxxx
   ```
   Опционально для «чата» создайте вторую форму и задайте:
   ```env
   VITE_CHAT_FORM_ENDPOINT=https://formspree.io/f/yyyyy
   ```
3. Соберите: `npm run build`, задеплойте `dist`.

В этом режиме основная заявка и (если задали) чат уходят в Formspree, свой API не нужен.

---

## Кратко

| Ситуация | Действие |
|----------|----------|
| Есть свой API | В `.env` при сборке: `VITE_API_BASE_URL=https://...` и CORS на бэкенде для домена сайта. |
| Нет API | В `.env` при сборке: `VITE_CONTACT_FORM_ENDPOINT=https://formspree.io/f/...` (и при желании `VITE_CHAT_FORM_ENDPOINT`). |

Переменные `VITE_*` подставляются в код **в момент сборки**. После `npm run build` менять их на хостинге бесполезно — нужно пересобрать с нужным `.env`.
